# UFO Galaxy 设备兼容性自检清单 (Device Compatibility Checklist)

**版本**: v1.0
**日期**: 2026-02-07

在尝试将新设备接入 UFO Galaxy 系统之前，请对照此清单进行自检。这能帮助您快速判断接入难度和可行性。

## 1. 物理连接层 (Physical Layer)
*您的设备能物理连接到运行 UFO Galaxy 的主机吗？*

- [ ] **USB**: 设备是否有 USB 接口？主机能否识别到 `/dev/ttyUSB*` 或 `/dev/hidraw*`？
- [ ] **网络 (LAN/Wi-Fi)**: 设备是否与主机在同一局域网？能否 Ping 通设备的 IP？
- [ ] **蓝牙 (Bluetooth)**: 主机是否有蓝牙适配器？`hcitool scan` 能否扫描到设备的 MAC 地址？
- [ ] **串口 (Serial/RS485)**: 是否有 USB 转串口适配器？波特率是否已知（如 9600, 115200）？

> ✅ **判定**: 只要物理上能连通，软件上就有 90% 的概率能搞定。

## 2. 协议文档层 (Documentation Layer)
*您知道设备说的是什么语言吗？*

- [ ] **标准协议**: 设备是否使用标准协议？(MQTT, HTTP, RTSP, ONVIF, Modbus, Zigbee ZCL) -> **难度: 低**
- [ ] **公开文档**: 厂商是否提供了通信协议文档（PDF/Wiki）？ -> **难度: 中**
- [ ] **SDK/Demo**: 厂商是否提供了 Python/C/Java 的示例代码？ -> **难度: 低**
- [ ] **无文档 (黑盒)**: 没有任何文档，只有设备本身。 -> **难度: 高 (需使用 AI 协议解析)**

## 3. 软件驱动层 (Driver Layer)
*操作系统能识别这个硬件吗？*

- [ ] **Linux 驱动**: 在 Linux 下运行 `lsusb` 或 `dmesg` 能看到设备名吗？
- [ ] **无需驱动**: 设备是标准的 HID (键盘/鼠标) 或 CDC (串口) 设备吗？
- [ ] **专有驱动**: 是否需要安装厂商提供的 `.ko` 内核模块？(UFO Galaxy 运行在用户态，内核驱动需自行安装)

## 4. 安全与加密层 (Security Layer)
*设备是否拒绝与陌生人说话？*

- [ ] **无加密**: 通信数据是明文的（如 ASCII 字符串）。 -> **完美**
- [ ] **标准加密**: 使用 TLS/SSL，且允许导入自签名证书。 -> **可行**
- [ ] **专有加密**: 通信数据是一堆乱码，且厂商未公开解密算法。 -> **困难 (AI 可能也无法破解强加密)**
- [ ] **云端绑定**: 设备必须连接厂商云端才能工作，不支持局域网控制。 -> **极难 (需破解或抓包)**

## 5. 接入方案推荐

根据您的勾选结果，推荐以下接入方案：

| 场景 | 推荐方案 | 示例代码 |
| :--- | :--- | :--- |
| **标准网络设备** | 直接配置 IP/Port | `unified_launcher.py` 配置 |
| **有文档的串口/蓝牙** | 编写 Protocol Node | 继承 `BaseProtocolNode` |
| **无文档但无加密** | AI 协议解析 | 启用 `ai_parse_unknown_protocol` |
| **强加密/云端绑定** | 放弃直连，尝试云对接 | 调用厂商 Cloud API (HTTP) |
